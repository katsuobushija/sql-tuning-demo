AI請求書自動仕訳システム

AWS Lambda Terraform Bedrock 請求書PDFからAIが勘定科目を自動判定するサーバーレスシステム。
経理担当者が手作業で行っていた請求書の仕訳作業を自動化するシステムです。請求書（PDF/Excel）をアップロードすると、Amazon Bedrock（Claude）が内容を読み取り、勘定科目を自動判定します。AWS Lambda を中心としたサーバーレス構成で、インフラ管理不要・低コストで運用できます。

▸ 処理フロー

  • 1. アップロード：Streamlit UIから請求書ファイル（PDF/Excel）をS3へアップロード
  • 2. AI抽出：S3トリガーでLambdaが起動、Bedrock（Claude Haiku）が請求書の内容を構造化データとして抽出（取引先名・日付・金額・税額・明細）
  • 3. 勘定科目の判定：抽出データをもとにAIが最適な勘定科目（仕入高・外注費・通信費・消耗品費 等）を推定
  • 4. 確認・修正：経理担当者がAIの判定結果を確認し、必要に応じて修正・承認
  • 5. CSV出力：確認済みデータを会計ソフト用のCSVとしてエクスポート

▸ アーキテクチャ

Streamlit UI
    ↓ presigned URL
Amazon S3（請求書ファイル）
    ↓ S3 Event Trigger
AWS Lambda（extract_invoice）
    ↓
Amazon Bedrock（Claude Haiku）
    ↓ 構造化データ
Amazon DynamoDB（仕訳データ保存）
    ↓
API Gateway → Lambda（search / update / export）
    ↓
Streamlit UI（確認・修正・CSV出力）

▸ 技術スタック

  • コンピュート：AWS Lambda（Python 3.12）× 5関数
  • AI/ML：Amazon Bedrock（Claude Haiku）─ 請求書内容の抽出と勘定科目推定
  • ストレージ：Amazon S3（ファイル保管）+ DynamoDB（仕訳データ、GSI 2本）
  • API：API Gateway（REST）
  • フロントエンド：Streamlit（Python製Web UI）
  • IaC：Terraform（モジュール構成：storage / compute / data / api）
  • CI/CD：GitHub Actions（lint + Terraform validate）

▸ 設計のポイント

  • LLMによる柔軟な対応：PDF・Excelなど多様なフォーマットの請求書を、ルールベースではなくAIが読み取るため、新しいフォーマットにもコード変更なしで対応
  • サーバーレス構成：EC2のような常時稼働サーバーが不要。月額¥700〜2,000程度（500件処理時）の低コスト運用
  • 人による確認・承認フロー：AIの判定を経理担当者が確認・修正する仕組みで、精度と信頼性を両立
  • Terraform による再現性：インフラ全体をコードで管理し、環境の再構築・移行が容易
